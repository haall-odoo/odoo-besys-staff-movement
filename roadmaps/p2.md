# SysAdmin workflows

# Type embedded in Odoo and reflect customization
Types are now embedded in Odoo and can be created / modified.
Field in movement types allows administrator to choose to make appear some part of the form of a staff movement.
For example if a type has the checkbox *Is equipment needed* ticked , every staff movement of this type will display the equipment boolean field.
Another example: if a type has the checkbox *Is former employee link needed* ticked, every staff movement of this type will display the many2one employee field


# Model for task admin
as create backpack / setup laptops etc
Two models have been created
- staff.movement.task.template
- staff.movement.task
  
The model staff.movement.task.template permit administrator to create task template, which can be linked to a staff.movement.type to generate a list of staff.movement.task in the staff.movement record selecting the staff.movement.type.
I know it is a bit unclear so for example:
The type 'hired' is linked to the tasks 'prepare backpack', 'create user' and 'install laptop'.
The type 'change of position' is linked to the task 'reset access rights' and 'install laptop' (just an example).
If the type selected for the staff.movement XYZ is 'hired' the task list will be updated to 'prepare backpack', 'create user' and 'install laptop'.
If the type selected for the staff.movement ABC is 'change of position' the task list will be update to 'reset access rights' and 'install laptop'.
It ensures a task integrity and no collission while a same task template is used among several staff.movement.type or a same type is shared by several staff.movement.
