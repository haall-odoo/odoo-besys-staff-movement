# Phase 1: Proof of Concept
## Introduction
This module is intended ot extend the HR module to internalise *arrivals*, *change of positions/offices* and *departures*.
The first focuses are on an integration of *staff.movement* module into HR and understand how works the Odoo's Python framework.

After the phase 1 it should provide:
- A simple list view of the staff.movements records 
  - Grouped by default on *movement type*
  - Open forms *staff.movement.form*
- Add a button in the Employee app menu to access the above list view
- Add a batch action to fire employee → should let chose the date
- Add a button in Employee records to create a movement
- Add a page in the employee record notebook to track their history


## Technically
- Extension of model hr.employee
- Addition of model staff.movement
- Addition of two wizard (default and departure)
